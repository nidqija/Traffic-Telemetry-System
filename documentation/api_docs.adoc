== Introduction

This documentation covers the full localized application architecture for the Traffic Light Monitoring System. The application consists of a Streamlit-based graphical user interface linked directly to an internal Text-to-SQL data pipeline. Instead of using web APIs or serial protocols, users query an embedded SQLite instance containing hardware pin states using natural language.

== System Architecture

The software stack follows a decoupling pattern across two distinct areas:
. **The Presentation Layer (Streamlit Dashboard):** Manages the user inputs, state management, execution feedback, and query visualization.
. **The Programmatic Analytical Backend (Factory Pattern):** Handles the prompt orchestration, parsing layers, external inference interactions, and local hardware database executions.

=== Database Schema Reference
Analytical routines query a local SQLite file (`traffic_data.db`) that maps physical micro-controller hardware pins to data logs:

* **Table Name:** `traffic_data`
* **Columns:**
** `id` (INTEGER, Primary Key): Unique row identifier.
** `timestamp` (TEXT): ISO-8601 formatted date-time record.
** `pin_12` (INTEGER): Red light state tracking flag (`1` = ON, `0` = OFF).
** `pin_7` (INTEGER): Yellow light state tracking flag (`1` = ON, `0` = OFF).
** `pin_8` (INTEGER): Green light state tracking flag (`1` = ON, `0` = OFF).

== User Interface Layer (Streamlit Application)

The dashboard provides an entry point for natural language interaction, displaying generated queries along with execution metrics.

=== Main Application Component: `ai_agent_page()`
* **Purpose:** Sets up page metadata configurations, embeds custom CSS aesthetics, renders sample question suggestions, captures text queries, triggers background processing hooks, and surfaces reactive code feedback blocks.
* **Initialization Workflow:**
. Safely requests an initialized worker via `AiFactoryManager.get_agent()`.
. Gracefully errors out and halts thread processing (`st.stop()`) if runtime secrets or environment assets are missing.
. Watches user submit button actions to dynamically spin up operational loading overlays (`st.spinner()`).

== Programmatic Core Interfaces

### Base Class: `AiAgent`
The foundational abstract layer responsible for contextual database schema definitions and output string normalization.

#### `get_schema_context()`
* **Description:** Houses the declarative constraints, table properties, and formatting requirements passed along to AI systems.
* **Return Value:** `str` (A formatted ruleset block).

#### `clean_sql(sql_string)`
* **Description:** Strips away LLM-generated markdown wrapper artifacts (e.g., ` ```sql ` blocks or loose syntax flags) to guarantee a raw, compile-ready executable string.
* **Parameters:**
** `sql_string` (`str`): Unprocessed text directly output by an AI worker.
* **Return Value:** `str` (A purified SQLite query execution string).

---

### Concrete Class: `CloudflareWorkerFactory`
Extends `AiAgent` to coordinate remote-inference execution patterns, payload generation, and local database interactions.

#### `__init__(model_name)`
* **Description:** Verifies and loads operational platform credentials (`CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_API_TOKEN`) and flags the target model engine.
* **Parameters:**
** `model_name` (`str`, optional): Targets the specialized AI engine. Defaults to `@cf/meta/llama-3.2-3b-instruct`.

#### `ask_questions(question)`
* **Description:** Core translation method. It builds system contexts, submits a payload to Cloudflare Workers AI, cleans the resultant string using internal text processing helpers, executes it against `traffic_data.db`, and parses database schema data.
* **Parameters:**
** `question` (`str`): A natural language question provided by the interface (e.g., *"What is the latest traffic record?"*).
* **Return Values:** ** On Success: `tuple[list, list]` -> `(query_results_rows, table_column_headers)`
** On Failure: `str` -> Error diagnostic string detailing the network or payload structural schema exception.
